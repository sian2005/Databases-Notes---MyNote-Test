View

A premade Query that you can use in other queries

Views exist in two types, virtual and materialized. Virtual stores the query in memory, and materialized stores the table result.

  • Usage - Views can be accessed as tables
  • Simplify - Views can simplify data access

Example

PC_A_PriceList (model, price) is a view "containing" the price of all PC made by maker A: 

CREATE VIEW PC_A_Pricelist AS
SELECT model, price
FROM PCs
WHERE model IN (
    SELECT model
    FROM Products
    WHERE maker = 'A'
)

SELECT AVG (price)
FROM PC_A_Pricelist
Usage in a Query

Find the average price of a PC made by maker A. 

SELECT AVG (price)
FROM PC_A_PriceList;
